# Foxit PDF SDK for Web 示例 - Vue.js

本节介绍Vue.js的两个示例。一个介绍如何快速运行Foxit PDF SDK for Web包中的样板示例，另外一个介绍如何将Foxit PDF SDK for Web集成到现有的Vue应用程序中。

# 快速运行Vue.js样板示例

Foxit PDF SDK for Web为 @vue/cli (opens new window) 应用程序提供了一个样板工程。该工程位于Foxit PDF SDK for Web包中的 integrations/ 目录下。

# 先决条件

  • Nodejs (opens new window) 和 npm (opens new window)
  • Foxit PDF SDK for Web (opens new window)

# 开始

在命令行中进入Foxit PDF SDK for Web的 ../integrations/vue.js/ 目录，执行：

npm run setup

此设置将实现以下操作：

  • npm install – 安装依赖项。
  • npm run update-sdk
    • 将Foxit PDF SDK for Web包中的 lib 文件夹拷贝到 ../integrations/vue.js/src/，并且自动重命名为foxit-lib。
    • 将 ..examples/license-key.js 拷贝到 ../integrations/react.js/src/。

# 运行示例

npm run serve

现在已经准备好启动应用程序了。打开浏览器，导航到 <http://127.0.0.1:9103/> 加载您的应用程序。

# 集成 Foxit PDF SDK for Web到现有的Vue.js工程

假设您已经安装了默认设置的 @vue/cli (opens new window) 应用程序。

# 先决条件

  • Nodejs (opens new window) 和 npm (opens new window)
  • @vue/cli (opens new window)
  • Foxit PDF SDK for Web (opens new window)

# 设置

为了方便后面介绍，我们将现有工程的根文件夹称为 VueJS，将Foxit PDF SDK for Web根文件夹称为SDK。

  1. 在SDK文件夹下，将下列文件夹和文件拷贝到 VueJS/src 目录下，并将 lib 文件夹重命名为 foxit-lib。

    • lib文件夹。
    • ../examples/license-key.js 文件。
    • /integrations/vue.js/src/preload.js 文件。

    另外，为了正确引用字体库，您还需要将SDK目录下的 external 文件夹拷贝到 VueJS/src/foxit-lib/assets。

  2. 运行 npm i -D cross-env 安装 cross-env，然后在 VueJS/package.json 文件中的 serve 和 build 部分添加如下的代码段：

    cross-env NODE_OPTIONS=--max_old_space_size=8192 NODE_ENV=development vue-cli-service serve
cross-env NODE_OPTIONS=--max_old_space_size=8192 NODE_ENV=production vue-cli-service build

    此步骤的目的是避免内存泄露的错误。

# 配置

在VueJS中添加如下文件的配置：

  • vue.config.js
  • .eslintignore

有关配置的详细信息，请参阅SDK目录下的 integrations/vue.js/counterpart 文件。

# 组件

  • 在VueJS中创建 src/components/PDFViewer.vue，然后在 src/App.vue 中引用它。

有关配置的详细信息，请参阅SDK目录下的 integrations/vue.js/counterpart 文件。

# 引用 Addons

如果您正在将Foxit PDF SDK for Web集成到现有的Vue工程中，您需要先阅读本章节内容。有关Addons更详细的介绍，请查阅Addons。

在这里，我们介绍三种在Vue工程中引用SDK addons的方法，您可以根据需要选择其中的一种。下面的 对比介绍将帮忙您更好地了解这三种方法的区别，并做出最佳的选择。

# 1. 引用碎片化的addons

此方法在7.2之前的版本中是默认使用的。您需要打开 PDFViewer.vue，然后引用addons，如下所示：

this.pdfui = new UIExtension.PDFUI({
    addons: [
        the_path_to_foxit_lib + '/uix-addons/file-property/addon.info.json',
        the_path_to_foxit_lib + '/uix-addons/full-screen/addon.info.json',
        // .etc
    ],
    // other options
});

其中，the_path_to_foxit_lib 是SDK目录下的lib 文件夹。

# 2. 导入模块化的addons

  1. 安装。

    npm i -D @foxitsoftware/addon-loader

  2. 更新 vue.config.js，您可以参阅/integrations/vue.js/vue.config.js。

  3. 在 PDFViewer.vue，为每个addon导入 addon.info.json：

        import * as UIExtension from '../foxit-lib/UIExtension.full.js'
    import filePropertyAddon from '../foxit-lib/uix-addons/file-property/addon.info.json';
    import multiMediaAddon from '../foxit-lib/uix-addons/multi-media/addon.info.json';
    import passwordProtectAddon from '../foxit-lib/uix-addons/password-protect/addon.info.json';
    import redactionAddon from '../foxit-lib/uix-addons/redaction/addon.info.json';
    import pathObjectsAddon from '../foxit-lib/uix-addons/path-objects/addon.info.json';
    import printAddon from '../foxit-lib/uix-addons/print/addon.info.json';
    import fullScreenAddon from '../foxit-lib/uix-addons/full-screen/addon.info.json';
    import importFormAddon from '../foxit-lib/uix-addons/import-form/addon.info.json';
    import exportFormAddon from '../foxit-lib/uix-addons/export-form/addon.info.json';
    import undoRedoAddon from '../foxit-lib/uix-addons/undo-redo/addon.info.json';
    import textObjectAddon from '../foxit-lib/uix-addons/text-object/addon.info.json';

    并将addons传递给PDFUI构造函数：

    const libPath = '/foxit-lib/';
    this.pdfui = new UIExtension.PDFUI({
        addons: [
            filePropertyAddon,
            multiMediaAddon,
            passwordProtectAddon,
            redactionAddon,
            pathObjectsAddon,
            printAddon,
            fullScreenAddon,
            importFormAddon,
            exportFormAddon,
            undoRedoAddon
        ].concat(
            // text-object addon is disabled on mobile platform
            UIExtension.PDFViewCtrl.DeviceInfo.isMobile
                ? []
                : textObjectAddon
        ),
        // other options
    });

# 3. 引用 allInOne.js

allInOne.js是所有addons的合集，位于 foxit-lib/uix-addons/ 目录下。引用该文件，打开 PDFViewer.vue，如下所示更新代码：

// ...
import * as UIExtension from '../foxit-lib/UIExtension.full.js';
import * as Addons from '../foxit-lib/uix-addons/allInOne.js';
// ...

然后传递参数给PDFUI构造函数：

this.pdfui = new UIExtension.PDFUI({
    addons: UIExtension.PDFViewCtrl.DeviceInfo.isMobile
        ? Addons.filter(it => it.getName() !== 'textEditObject')
        : Addons,
    // other options
});

# 加载方法对比

加载方法对比 配置 HTTP 请求 可修改的 (比如，修改本地化资源，和addon.info.json)
碎片化 无 n+ 支持修改
模块化 配置 gulp 0 支持修改，但是修改后需要重新合并addons
allInOne.js 无 1 不支持修改

备注：您可以使用我们的 Addons合并工具来重新编译allInOne.js。

# 运行应用程序

npm run serve

现在，一切都已设置完成。打开浏览器，根据您的设置输入地址来启动应用程序。
