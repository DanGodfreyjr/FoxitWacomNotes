# Foxit PDF SDK for Web 示例 - React.js

本节介绍React.js的两个示例。一个介绍如何快速运行Foxit PDF SDK for Web包中的样本示例，另外一个介绍如何将Foxit PDF SDK for Web集成到由WebPack 和 Babel创建的现有的React应用程序中。

# 快速运行React.js样本示例

备注：在下面的介绍中，Foxit PDF SDK for Web 的根文件夹被称为 root。

Foxit PDF SDK for Web为React应用程序提供了一个由WebPack 和 Babel创建的样板工程。该工程位于Foxit PDF SDK for Web包中的integrations/ 目录下。

# 概述工程结构

├─app/
│  ├─components/

# 先决条件

  • Nodejs (opens new window) 和 npm (opens new window)
  • Foxit PDF SDK for Web (opens new window)

# 开始

导航到 root/integrations/react.js/，然后执行：

npm run setup

此设置将实现以下操作：

  • npm install 安装依赖项。
  • npm run update-sdk
    • 将root文件夹下的 lib 文件夹拷贝到 root/integrations/react.js/app/，并且自动重命名为 foxit-lib。
    • 将 root/examples/license-key.js 拷贝到 root/integrations/react.js/app/ 目录下。

# 引用Addons

如果您正在将Foxit PDF SDK for Web集成到现有的React工程中，您需要先阅读本章节内容。有关Addons更详细的介绍，请查阅 Addons。

在这里，我们介绍三种在React 工程中引用SDK addons的方法，您可以根据需要选择其中的一种。下面的 对比介绍将帮忙您更好地了解这三种方法的区别，并做出最佳的选择。

# 1. 引用碎片化的addons

此方法在7.2之前的版本中是默认使用的。您需要打开 components/PDFViewer/index.js，然后引用addons，如下所示：

this.pdfui = new UIExtension.PDFUI({
    addons: [
        the_path_to_foxit_lib + '/uix-addons/file-property/addon.info.json',
        the_path_to_foxit_lib + '/uix-addons/full-screen/addon.info.json',
        // .etc
    ],
    // other options
});

其中，the_path_to_foxit_lib 是SDK目录下的lib 文件夹。

# 2. 导入模块化的addons

  1. 安装。

    npm i -D @foxitsoftware/addon-loader

  2. 更新 development/webpack/webpack.base.js 配置：

    {
    test: /addon\.info\.json$/,
    include: /foxit-lib/,
    use: [{
        loader: 'babel-loader',
        options: options.babelLoaderOptions
    }, '@foxitsoftware/addon-loader'],
    type: 'javascript/auto'
}

  3. 在 components/PDFViewer/index.js 文件中，为每个addon导入 addon.info.json：

        import * as UIExtension from '../foxit-lib/UIExtension.full.js'
    import filePropertyAddon from '../foxit-lib/uix-addons/file-property/addon.info.json';
    import multiMediaAddon from '../foxit-lib/uix-addons/multi-media/addon.info.json';
    import passwordProtectAddon from '../foxit-lib/uix-addons/password-protect/addon.info.json';
    import redactionAddon from '../foxit-lib/uix-addons/redaction/addon.info.json';
    import pathObjectsAddon from '../foxit-lib/uix-addons/path-objects/addon.info.json';
    import printAddon from '../foxit-lib/uix-addons/print/addon.info.json';
    import fullScreenAddon from '../foxit-lib/uix-addons/full-screen/addon.info.json';
    import importFormAddon from '../foxit-lib/uix-addons/import-form/addon.info.json';
    import exportFormAddon from '../foxit-lib/uix-addons/export-form/addon.info.json';
    import undoRedoAddon from '../foxit-lib/uix-addons/undo-redo/addon.info.json';
    import textObjectAddon from '../foxit-lib/uix-addons/text-object/addon.info.json';

    并将addons传递给PDFUI构造函数：

    const libPath = '/foxit-lib/';
    this.pdfui = new UIExtension.PDFUI({
        addons: [
            filePropertyAddon,
            multiMediaAddon,
            passwordProtectAddon,
            redactionAddon,
            pathObjectsAddon,
            printAddon,
            fullScreenAddon,
            importFormAddon,
            exportFormAddon,
            undoRedoAddon
        ].concat(
            // text-object addon is disabled on mobile platform
            UIExtension.PDFViewCtrl.DeviceInfo.isMobile
                ? []
                : textObjectAddon
        ),
        // other options
    });

# 3. 引用 allInOne.js

allInOne.js是所有addons的合集，位于 foxit-lib/uix-addons/ 目录下。引用该文件，打开 components/PDFViewer/index.js，如下所示更新代码：

// ...
import * as UIExtension from '../foxit-lib/UIExtension.full.js';
import * as Addons from '../foxit-lib/uix-addons/allInOne.js';
// ...

然后传递参数给PDFUI构造函数：

this.pdfui = new UIExtension.PDFUI({
    addons: UIExtension.PDFViewCtrl.DeviceInfo.isMobile
        ? Addons.filter(it => it.getName() !== 'textEditObject')
        : Addons,
    // other options
});

# 加载方法的比较

加载方法 配置 HTTP 请求 可修改的 (比如，编辑本地化资源，和addon.info.json)
碎片化 无 n+ 支持修改
模块化 配置 gulp 0 支持修改，但是修改后需要重新合并addons
allInOne.js 无 1 不支持修改

备注：您可以使用我们的 Addons合并工具来重新编译allInOne.js。

# 运行应用程序

在Shell中，运行如下的命令来启动开发服务：

npm start

现在已经准备好启动应用程序了。打开浏览器，导航到 <http://127.0.0.1:9102/> 加载您的应用程序。

# 编译

npm run build

编译生成的文件在 root/integrations/react.js/dist 目录下。

# 测试

cd ./dist && http-server -p 8080

# 集成 Foxit PDF SDK for Web到现有的React工程

假设您已经使用Webpack 和 Babel创建了一个React应用程序。

# 先决条件

  • Nodejs (opens new window) 和 npm (opens new window)
  • Reac.js created with WebPack and Babel (opens new window)
  • Foxit PDF SDK for Web (opens new window)

# Webpack 配置

这里，我们将现有的React工程根目录和 Foxit PDF SDK for Web 根目录分别称为ReactJS 和 SDK.

  1. 在 ReactJS/development/webpacK 文件夹下创建和配置如下的3个文件：

    • webpack.base.js
    • webpack.dev.js
    • webpack.prod.js

    有关配置详细信息，请参阅 SDK/integrations/react.js/development/webpack/ 目录下的counterpart文件。您也可以直接将文件复制到 ReactJS/development/webpacK。

  2. 在package.json文件中配置npm脚本。

"script": {
"start": "webpack-dev-server --config development/webpack/webpack.dev.js",
"build": "webpack --config development/webpack/webpack.prod.js"
}

# 添加依赖项和JS引擎预加载文件

  1. 将SDK目录下的 lib 文件夹和 SDK/examples/license-key.js 文件拷贝到ReacJS/app 文件夹下，并将lib重命名为foxi-lib。另外，为了正确引用字体库，您还需要将SDK目录下的 external 文件夹拷贝到 ReactJS/app/foxit-lib/assets。

  2. 使用ReacJS创建和配置以下文件：

    • babel.config.js (opens new window)
    • ../app/components/PDFViewer/index.js
    • ../app/containers/App/index.js
    • ../app/ 目录下的 index.html, app.js 和 preload.js

    有关配置详细信息，请参阅SDK目录下对应的文件。您也可以直接将这些文件拷贝到现有工程的counterpart文件夹下。

# 运行应用程序

npm run start

现在，一切都已设置完成。打开浏览器，导航到 http://127.0.0.1:9102/ (opens new window) 启动应用程序。
